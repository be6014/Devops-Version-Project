# DevOps Git Version-Controlled Project

## Objective
Manage a DevOps project using Git best practices.

## Tools
- Git
- GitHub

## Steps
1. Initialize repository:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   ```
2. Create branches:
   ```bash
   git branch dev
   git branch feature/login
   ```
3. Merge using PR workflow (GitHub).
4. Add .gitignore and tags:
   ```bash
   git tag v1.0
   ```

## Files
- app.py → sample application
- team.csv → dataset
- .gitignore → ignored files

## Outcome
Learn Git workflows: branching, commits, merge, tags.
